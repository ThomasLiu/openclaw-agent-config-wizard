# 对话模板与术语表

## OpenClaw 专有名词（人话版）

| 术语 | 人话解释 |
|------|----------|
| workspace | agent 的"工作间"——它的文件夹，里面有它的配置文件和记忆 |
| binding | "绑定"——把某个频道（微信/Telegram/Slack）接给特定 agent |
| heartbeat | "心跳"——定期轻量检查，比如看看有没有新邮件/日程（和 cron 分工） |
| session | "会话"——一次和 agent 的对话 |
| agentId | agent 的名字/标识符 |
| main | 默认 agent 的固定名字 |

## 问答话术模板

### 开场（用户说"帮我配一个做 X 的 agent"）

```
好的！我来帮你设计一个专门做 [X] 的 agent。

在开始之前，我需要确认几个问题，这样配出来的 agent 才真正好用：

1. [具体问题] 
   比如：A) 场景A  B) 场景B  C) 按你的情况改

2. [具体问题]
   ...
```

### 术语解释

当用户用了不懂的词，先解释再确认：

```
你说的 [术语]，我理解是指 [一句话解释]。典型场景是 [小例子]。
是这样吗？
```

### 产出具役草案

```
📋 以下是草案（确认后我来执行）：

**Agent ID**: xxx
**用途**: xxx  
**绑定**: xxx
**工作区**: ~/.openclaw/agents/xxx/

📁 文件结构：
- AGENTS.md
- SOUL.md
- IDENTITY.md
- TOOLS.md
- HEARTBEAT.md

---
有问题请说，确认后我执行。
```

## 垂直领域流程草稿法

对于用户说职位/流程但没细节的情况：

1. **先给定义**：一句话说清楚这个词在你理解里指什么
2. **再给场景**：典型一次任务从头到尾是怎样的
3. **问确认**：「是不是这样？」
4. **确认后才写进文件**

禁止：假装精通法律/医疗等受监管行业的结论，用「协助整理信息、提醒、草稿」等边界表述。
